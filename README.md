# Stats-Api Backend
The backend work being done for an NBA statistics platform as a personal project. 

Features a REST API built with FastAPI in order to establish endpoints with data being currently retrieved using the BallDontLie API. 
> Note: I'm currently in the process reconfiguring this program in order to use a PostgreSQL database for the presentation/manipulation of data, my ideas for the direction of this project have moved in a slightly different direction than what I originally envisioned. 

### Why no ORM?
Great question! I want to get better at writing SQL queries and improve my schema design skills, as well as work on my overall proficiency with relational DBs.
